测试文章图片显示

图片
Publish Date:   2022-12-18

今天顺利通过hexo d将网页文件推送至GitHub，网页正常显示，网页复制的图片正常显示，但是本地插入的图片未能加载成功。

网上搜索解决方案，如下

解决hexo中发布文章后图片无法显示的问题

1. 问题描述

默认情况下，在 hexo 中新建一篇markdown博文，在文章中插入本地图片，再部署到 github 上面，是无法显示这些图片的。

2. 解决方法

首先找到博客根目录下面的_config.yml文件，修改post_asset_folder的值为true.

接着 git bash 切换到博客根目录下面，安装第三方插件hexo-asset-image

接着hexo new "文章名" 新建一篇文章，会发现source/_posts/目录下面会生成一个和 markdown 文件同名的文件夹。再将当前这篇文章中需要插入的图片放到这个文件夹中，再到 markdown 中引用该文件夹中的图片，最后hexo g -> hexo d，就可以正常的查看这些图片了。

现在测试：新建了这篇文章，图片文件夹自动生成了。

修改图片根目录–>当插入本地图片时，复制到生成图片文件夹中

上述方法无法解决图片显示的问题，还是用的昨天的方法，设置图片根目录D:\New Blog\source，文件夹下建立images文件夹，设置当插入本地图片时，复制到该images文件夹（设置时，选复制到指定文件夹，输入../../source/images/${filename}，选择优先相对路径），然后插入图片，typora能显示，hexo s 正常显示，然后hexo g，最后hexo d。

上传到GitHub后，打开网页图片不显示，延迟的问题，等一会就可以了，因为再实验第二篇文章的时候，第一篇文章的图片已经加载出来了。

目前这样设置，可以解决图片三方显示的问题，但是之间具体的联系还是不太清楚，进一步摸索。
