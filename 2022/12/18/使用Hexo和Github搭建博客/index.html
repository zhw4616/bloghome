使用Hexo和Github搭建博客

Hexo
Publish Date:   2022-12-18

最近通过查找资料，建设了自己的博客，这期间历经了其实我感觉作为一个程序员来搞的话半天时间差不多就能搞定，最多一天时间。因为不懂，从零开始摸索，用了五天的时间，从使用GitHub Pages部署静态网页，到使用hexo在GitHub上部署主题博客，虽然教程简单，但是中间踩了很懂坑，有的坑一坑就是半天过不，今天基本完成了，后续就是美化和进入折腾了，目前运行正常。通过这篇文章简单回忆一下。感谢网络上提供帮助的人，主要根据作者Andy的博客地址：https://huahuipj.github.io/作为指导，遇到问题Google解决。这篇文章也是在原文基础结合自己的搭建总结的。

Next 1：

给大家介绍搭建的博客，是需要部署在Github上的，首先要有一个Github账号的，没有的小伙伴去注册一个吧，操作比较简单这里就不在多讲了。

Next 2：

需要大家安装一下Git，相信作为一个程序员大家都已经安装过了，没有安装过的网上搜一下，教程很多也比较简单。

（这里说明一下，因为单位和家里电脑都在进行搭建，单位电脑使用git最新版本2.39.0推送到GitHub失败，不明原因，后来网上搜索，换了git老版本解决问题。家里电脑使用git最新版本，正常使用。）

Next 3：

安装nodejs和Hexo，其实这个步骤是成功部署博客的核心。

  • 首先我们要安装一下nodejs，nodejs官网, 具体安装步骤。

  • 安装完nodejs后，我们就要安装Hexo了，Hexo就是我们的个人博客网站的框架。

  • 在安装之前，我们要先在GitHub上创建一个仓库，关于这个仓库，之前部署静态网站时已经建过，所以这里我新建了一个。第一次建仓库的话，建议就建，GitHub用户名.github.io，如下图所示。

    创建仓库

    仓库的名称格式为你的用户名.github.io，点击Create repository按钮后创建成功。

    注意：如果你没有创建过GitHub的token的话，这里最好创建一个，因为现在使用token来代替GitHub的密码进行登录，你之前的密码将不能用于GitHub的登录，我这里就因为这个原因浪费了半天时间。因为提交代码的时候无法用密码进行登录，报错403，然后我以为是我账户或密码有问题，最后查到是因为GitHub更换了验证方式[捂脸]。如何创建个人访问令牌

    （这里说明一下，我没有创建token，只是建立了SHH key，连接Git和GitHub。这一步是实现用git将文件推送到GitHub上面，因为我实现了推送，所以没再建token。我分别在单位和家里的电脑上创建了不同的SHH key，创建步骤见帮助文档https://docs.github.com/en/authentication）

    第一步是安装Hexo，首先在磁盘建立一个文件夹 Blog，（我在D盘根目录下建立文件夹，文件夹名字自己定，这里以Blog为例），点开Blog文件夹，鼠标右键打开 Git Bush Here，输入npm命令安装Hexo：
